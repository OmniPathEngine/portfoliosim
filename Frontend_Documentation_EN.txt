UI/UX DOCUMENTATION (ENGLISH)
ETF PORTFOLIO SIMULATOR (VERSION 5.0)

This document describes the structural User Interface (UI) architecture as well as the interactive User Experience (UX) patterns of the application.


CHAPTER 1: USER INTERFACE (UI) & EDUCATIONAL ARCHITECTURE

- INTRODUCTION TO THE WORLD OF ETFS
  The onboarding element ('What exactly are ETFs?') utilizes easy-to-understand, educational language. 
  Users learn the fundamental rules of the Buy & Hold approach (10 to 15 years). The visual design, 
  encapsulated in a separate 'bg-slate-800' container, detaches this learning segment from the rest of the 
  metrics, ensuring beginners are not immediately overwhelmed by numbers.

- EXPLANATION OF TERMINOLOGY (ETF JARGON)
  A responsive CSS grid systematically arranges complex terminology such as World Index, Emerging Markets, 
  ACWI, IMI, TER, and Acc. Subtle color accents ('text-blue-400', 'text-emerald-400', etc.) 
  support visual learning and group the concepts cognitively into logical boundaries.

- PORTFOLIO CONCEPTS
  The display of the four core strategies (Pro-Mix, Classic, All-in-One, Relaxed Entry) is dynamically 
  constructed via the JS function `renderConcepts()`. Each concept is rendered as an isolated card component. 
  The header is dominated by static metrics such as Ø Return, Volatility, contained companies, 
  and Ø TER, which are automatically extracted and aggregated from JSON objects ('blendedCagr', 'blendedVol').

- ACTIONABLE RECOMMENDATIONS & RISK WARNINGS
  Educational guidance is crafted through color-coded typography. Recommendations brightly glow in green/blue 
  ('text-emerald-400', 'text-blue-400'), indicating the target demographic for the strategy. 
  Hidden systemic risks or cluster risks (e.g., US-Tech dominance) are marked in signal red ('text-rose-400') 
  and supplemented with a warning icon.

- GRAPHICAL REPRESENTATION (CHART.JS)
  Via the `initPieChart()` function, seamless doughnut charts are pumped into the UI to map the 
  Regional, Sectoral, and Currency distributions (allocation). Long-term progression 
  is additionally rendered as a line chart. It illustrates the invested capital (gray), 
  along with the best-case, worst-case, and the heavily emphasized median trajectory of the respective portfolio. 
  All graphs are fully bound responsively to the browser window geometry. To strictly prevent visual clipping 
  across narrow pie slices, all interior metric labels and hover tooltips are natively sanitized utilizing `Math.round()`.

- MOUSEOVER TOOLTIPS
  Instead of rigid title attributes, the frontend utilizes an adaptive, CSS-driven tooltip system based 
  on Tailwind classes 'group-hover' (for desktop mice) and 'touch-hover' 
  (for mobile devices). This logic injects explanations for abstract metrics by fluidly fading in 
  floating containers ('absolute', 'z-50') out of thin air ('opacity-100').

- UNDER THE HOOD MODAL
  Deep mathematical revelations are concealed within a modal overlay ('#math-modal'). 
  This keeps the main page extremely clean (Progressive Disclosure). The modal contains 14 separate 
  blocks (rendered with KaTeX), which detail the stochastic processes and code architecture 
  of the OmniPath 5.0 Engine (Hawkes, Heston, QuickSelect, SFC32).


CHAPTER 2: USER EXPERIENCE (UX) & INTERACTION FLOW

- LOCALIZATION (GERMAN / ENGLISH)
  The application fluently speaks two languages. The function `setLang(lang)` handles the 
  immediate language toggle. CSS classes such as '.lang-de' and '.lang-en' are toggled 
  via JS ('classList.toggle') seamlessly. An important detail: Upon switching to English (USD), 
  the existing Euro inputs for initial capital and savings rate are live-converted via current exchange rates.

- DYNAMISCHE WECHSELKURSE
  To ensure accurate and consistent visual metric conversions, the asynchronous function 
  `fetchExchangeRate()` communicates directly with the external API 'open.er-api.com'. The currency conversion 
  rate from EUR to USD is silently fetched in the background up front, allowing the UI to localize 
  accurate global portfolio values continuously.

- INPUT-HANDLING (STICKY CONTROL PANEL)
  The central input field (initial capital, savings rate, escalation) is monitored by an IntersectionObserver 
  (Anchor: '#sticky-sentinel'). As soon as the user scrolls downwards, the panel safely locks 
  into place at the top bounds of the screen ('is-sticky'). 
  User keystrokes are filtered by the function `formatNumberInput` following strict Regex rules, 
  ensuring only pure financial strings are accepted. Superimposed is a smart timeout-debounce logic 
  (`debouncedUpdate()`), buffering the heavyweight calculation kernel over a 1000ms delay to 
  actively prevent CPU locking during rapid multi-keystroke inputs.

- INFLATION TOGGLE
  A physical toggle switch ('#inflation') enables immediate switching between nominal values 
  and real purchasing power (Fisher Present-Value discounting). Native to its 'change' event, 
  the checkbox enforces an immediate re-evaluation ('debouncedUpdate()'), heavily injecting economic 
  depth across all drawdowns and terminal asset trajectories.

- EXPANDABLE ADDITIONAL INFO (ACCORDION)
  The detailed allocation structure (regional, sectoral, and currency doughnut diagrams) is collapsed by default 
  to prevent cluttered UIs. Meanwhile, the distribution bar charts (Developed/Emerging Markets, Company Sizes) 
  have been structurally relocated to permanently render above the ETF weighting box for zero-interaction visibility.
  The JS trigger `toggleDetails(id)` wraps the accordion element, softly rotating 
  the chevron SVG icon ('rotate-180') via CSS transitions, additionally dispatching a manual 
  'resize' event to the global Window node to bypass annoying canvas compression bugs native to Chart.js.

- INSTITUTIONAL REFERENCES
  A subtle, dark-mode compliant footer embeds the logos and URLs of the four major financial data sources 
  (MSCI, BlackRock, J.P. Morgan, Vanguard). A CSS 'hover:-translate-y-1' reactively lifts the icons 
  against mouse boundaries, reinforcing professional institutional confidence in the simulated underlying metrics.

- FACTSHEET DEEP-LINKS
  Inside the rendered concept grids, each exact ETF features a direct hyperlink mapped to its respective 
  factsheet on 'justetf.com'. These anchor tags are constructed highly dynamically during JS loop iterations 
  utilizing their underlying ISIN identifiers combined with the active localization string ('justEtfLang'), 
  eliminating requirements for manual external due diligence research.
